# W2 Notes（每日 5 要点 + 3 个坑 + 1 个模板）

## Day 8（while / do-while / for：选择最合适的循环）

### 5 条要点
- `while`：先判断条件，可能一次都不执行
- `do-while`：先执行一次，再判断条件，至少执行一次
- `for`：计数场景最清晰（初始化/条件/迭代写在一行）
- 边界差异：`i < n` 产生 \(0..n-1\)；`i <= n` 产生 \(0..n\)
- 避免死循环：每个循环都要能指出“退出条件”和“让条件变化的语句”

### 3 个坑
- `i < n` / `i <= n` 写错会少算/多算一个元素（尤其是 1..n 求和）
- 平均值用 `int` 容易整除截断：`sum / n` 需要 `double` 或强转
- 不做输入校验：输入字母会导致解析失败/程序异常（本周统一用安全读取）

### 1 个模板（输入直到合法：do-while）

```java
int x;
do {
    x = in.readInt("输入整数（0-100 才能结束）：");
} while (x < 0 || x > 100);
```

## Day 9（嵌套循环：打印、枚举与计数）

### 5 条要点
- 外层循环通常控制“行/轮次”，内层循环控制“列/每行内容”
- 写图形题先想每一行要打印什么（空格数、星号数），再写循环
- 复杂循环别硬写：先画 3 行样例找规律（i=1/2/3）
- 判断质数只需试到 \(\sqrt{n}\)，枚举 1..n 质数个数用 `isPrime` 抽方法更清晰
- 需要对齐输出时，可以用 `\t` 或控制空格

### 3 个坑
- 内层循环边界写错（`j<=i` vs `j<i`）图形就会缺一列/多一列
- 在内层循环里写了 `println` 会导致每个字符都换行（输出乱）
- 质数判断忘记处理 2、偶数，或 `d*d` 溢出（这里用 int 且范围受控）

### 1 个模板（嵌套循环打印矩形边框）

```java
for (int i = 1; i <= rows; i++) {
    for (int j = 1; j <= cols; j++) {
        boolean border = (i == 1 || i == rows || j == 1 || j == cols);
        System.out.print(border ? "*" : " ");
    }
    System.out.println();
}
```

## Day 10（break / continue：控制流程但不写乱）

### 5 条要点
- `break`：跳出**当前**循环（结束这层循环）
- `continue`：跳过本轮剩余代码，进入下一轮
- `continue` 常用来“先过滤坏情况”，减少 `if` 嵌套层级
- 嵌套循环中 `break` 只跳出一层；要跳多层需要改结构/加标记（后面再说）
- 统计平均值要注意：`sum / count` 需要 `double`，避免整除截断

### 3 个坑
- 把 `break` 放错位置导致循环提前结束（比如放在错误分支外）
- `continue` 之后忘记更新某些变量可能造成死循环（尤其是 `while`）
- `count==0` 时直接算平均会除 0，先判断再算

### 1 个模板（continue 简化输入校验）

```java
try {
    v = Integer.parseInt(line);
} catch (NumberFormatException e) {
    continue; // 非法输入，直接跳过本轮
}
if (v < 0 || v > 100) continue;
// v 有效，进入统计
```

## Day 11（一维数组：存数据 + 遍历 + 统计）

### 5 条要点
- 数组是“定长容器”：创建后长度固定，用 `arr.length` 获取长度
- 读入数据后再统计：max/min/sum/avg/count 都是“遍历一遍数组”
- 遍历方式：普通 `for`（需要索引时） vs 增强 `for`（只看元素值时）
- 平均值避免整除：`avg = sum / (double)n`
- 原地反转用“双指针”从两端向中间交换，直到相遇

### 3 个坑
- 访问越界：有效索引是 `0..length-1`
- 统计初始化位置不对：例如把 `max` 初始化成 0 会影响全负数场景（通用写法是 `max=arr[0]`）
- 反转时交换次数过多：只需要交换到中点（`l<r`）

### 1 个模板（线性查找：找不到返回 -1）

```java
for (int i = 0; i < arr.length; i++) {
    if (arr[i] == target) return i;
}
return -1;
```

## Day 12（二位数组：表格数据与行列统计）

### 5 条要点
- 二维数组是“数组的数组”：`arr.length` 是行数，`arr[i].length` 是第 i 行列数
- 遍历二维数组：外层遍历行 `i`，内层遍历列 `j`
- 行统计（每班平均）：固定行 i，遍历 j 求和
- 全局统计（最高分位置）：双重循环同时记录“最大值 + 行列下标”
- 每行长度可能不同（锯齿数组），写循环时内层一定用 `arr[i].length`

### 3 个坑
- 把“行数”当“列数”用：`arr.length` 不是列数
- 内层循环写死列数，遇到锯齿数组会越界
- 平均值整除：`sum / count` 要转成 `double`

### 1 个模板（扫描最高分及位置）

```java
int top = arr[0][0], topI = 0, topJ = 0;
for (int i = 0; i < arr.length; i++) {
    for (int j = 0; j < arr[i].length; j++) {
        if (arr[i][j] > top) { top = arr[i][j]; topI = i; topJ = j; }
    }
}
```

## Day 13（方法：参数/返回值/重载/拆分职责）

### 5 条要点
- 方法签名 = 方法名 + 参数列表 + 返回值类型（决定“怎么调用”）
- 拆方法的典型信号：重复、过长、一个方法做了太多事（读输入 + 统计 + 打印）
- “统计类方法”常见写法：传入数组，返回计算结果（`max/min/avg/count`）
- 方法命名要体现“做什么/返回什么”（如 `countFail(scores, 60)`）
- 重载 = 同名不同参（参数个数/类型不同），编译器会按实参选择最匹配的版本

### 3 个坑
- 方法里直接依赖外部变量（全局/静态）会降低复用性，优先通过参数传入
- `avg` 如果返回 `int` 会丢小数，统计时用 `double`
- 重载不能只靠返回值区分（参数列表必须不同）

### 1 个模板（遍历数组求 max）

```java
int m = arr[0];
for (int i = 1; i < arr.length; i++) {
    if (arr[i] > m) m = arr[i];
}
return m;
```

## Day 14（大作业：StudentScoreManager）

### 5 条要点
- 分层思路：`Main` 只管菜单与交互，`Service` 管业务，`Student` 管数据结构
- 先做输入校验再做业务：id 必须唯一，score 必须 0–100，非法输入不崩溃
- CRUD 最常用的基础能力：按 id 查找（找不到就提示）
- 统计分析就是“遍历集合一遍”：sum/avg/max/min/failCount
- 代码可持续迭代的关键：把“查找/统计/校验”封装成方法，避免重复

### 3 个坑
- 平均值整除：`avg = sum / (double) n`
- 删除元素遍历时要小心（这里用索引遍历后 `remove(i)` 立即 `return`）
- 不处理空数据：列表为空、统计为空时要给出提示

### 1 个模板（按 id 查找）

```java
for (Student s : students) {
    if (s.getId() == id) return s;
}
return null;
```
