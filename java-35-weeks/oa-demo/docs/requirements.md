# OA 请假系统 · 需求摘要（W19 Day127）

> **技术路线（已定）**：**方案 A** — `oa-demo` 采用 **Maven Web（Servlet + JDBC / MyBatis）**；**Spring Boot 延后**，与课表「先跑通业务与数据」一致。

---

## 1. 角色（最小集合）

| 角色 | 说明 |
|------|------|
| **员工** | 登录、提交请假、查看**本人**请假单与状态、（可选）撤回未决单 |
| **部门经理** | 登录、查看待本部门审批的请假、**通过 / 驳回**、填写审批意见 |
| **系统管理员** | 用户/角色/权限维护、部门与员工主数据、审计查询（首版可弱化 UI，以接口为主） |

> 课表中的 **HR** 可在后续迭代并入「经理」或单独角色；首版不阻塞闭环。

---

## 2. 请假单状态（状态机 · 文字版）

首版实现重点：**`SUBMITTED` → `APPROVED` / `REJECTED`**；其余状态预留。

```
                    ┌──────────────┐
                    │   DRAFT      │  草稿（可选：保存未提交）
                    └──────┬───────┘
                           │ 提交
                           ▼
                    ┌──────────────┐
         ┌─────────│  SUBMITTED   │─────────┐
         │         └──────┬───────┘         │
         │ 经理驳回        │ 经理通过        │ 员工取消（规则内）
         ▼                 ▼                 ▼
  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
  │  REJECTED    │ │  APPROVED    │ │ CANCELLED    │
  └──────────────┘ └──────────────┘ └──────────────┘
```

**60 秒口述版**：员工提交后进入 **已提交**；经理只能把已提交改为 **已通过** 或 **已驳回**；通过后流程结束；驳回后员工可看到原因（后续可扩展「修改再提交」）。

## 4. 权限矩阵（接口级 · 与 RBAC 对齐方向）

| 资源/行为 | 员工 | 经理 | 管理员 |
|-----------|:----:|:----:|:------:|
| `POST /api/login` | ✓ | ✓ | ✓ |
| `POST /api/leaves`（提交） | ✓ | — | — |
| `GET /api/leaves/me` | ✓ | — | — |
| `GET /api/leaves/pending`（待审） | — | ✓ | — |
| `POST .../approve`、`.../reject` | — | ✓ | — |
| `/api/admin/**` | — | — | ✓ |

首版「数据范围」：**员工仅本人数据**；**经理本部门待审**（实现上可先简化再收紧，见 W19 后续日）。

## 5. 非功能（本周约束）

- 密码落库：**MD5 + salt**（课纲要求；生产更推荐 BCrypt，在实现类注释中注明即可）。
- 审计：每次状态变更写入 **`leave_actions`**（谁、何时、动作、备注），见 Day128 表设计。
