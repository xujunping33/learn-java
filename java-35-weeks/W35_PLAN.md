# 第 35 周学习计划：面试与就业指导（高频题 + 项目串讲 + 手写/原理）

> 对应总纲：**第35周（高频面试题详解）**  
> 目标：把你做过的项目与基础知识 **串起来**，能 **讲清楚**，并且能对高频题 **手写代码 / 讲清原理**。  
> 交付物：**面试题清单（你自己的答案）** + **项目亮点总结**（缓存、权限、事务、并发、性能优化）。  
> 时间：约 **2 小时/天**；本周 **Day 239 ~ Day 245**。

---

## 本周总节奏（固定模板，每天照抄）

- **25 min**：复习昨天写的答案（口述一遍，录音可选）
- **60 min**：当日主题：做题 / 写答案 / 改简历要点
- **20 min**：把 1 道题的答案整理进 `INTERVIEW-QA.md`（你自己的话）
- **15 min**：把当天收获写进 `PROJECT-HIGHLIGHTS.md`（只加 3～5 条 bullet）

## Day 239（周一）— 盘点与故事线：STAR + 「我最强的 2 个项目」

**做什么**

1. 列出你 Git 上 **能演示的仓库/模块**（名称 + 一键启动方式）。
2. 写 **2 条 STAR 故事**（各 200～300 字）：
   - **S** 背景：要解决什么问题  
   - **T** 你的任务  
   - **A** 你做了什么（技术选型 + 关键设计）  
   - **R** 结果（性能、可维护性、可观测、线上问题解决——实在没有线上就说「可复现/可测试」）
3. 定 **3 个「绝不翻车的自我介绍标签」**（例如：Spring Boot 后端、MySQL、微服务入门）。

**验收**

- 你能 **90 秒**讲完一个项目，不停顿找词。

## Day 240（周二）— Java 基础 & 集合 & 并发（高频手写 + 原理）

**学什么（按优先级）**

- `HashMap`：put 流程、扩容、hash 冲突；**为什么并发下不安全**
- `ConcurrentHashMap`：分段/桶锁思路（讲清版本差异不必背细节）
- 线程池：`ThreadPoolExecutor` 参数、拒绝策略、为什么不用无限队列
- `synchronized` / `volatile` / `JMM`（能讲可见性与有序性即可）
- 死锁四要素与如何避免

**做什么**

1. 手写：**生产者消费者（阻塞队列版）** 或 **LRU（LinkedHashMap 或手写链表版）** 二选一  
2. 整理 10 道「口述题」答案进 `INTERVIEW-QA.md`（每题 8～12 行）

**验收**

- 随意抽 3 题，你能 **不看稿**讲 2 分钟以上。

---

## Day 241（周三）— Spring / Spring Boot（IoC、AOP、事务、Web）

**学什么**

- IoC/DI：Bean 生命周期（能讲 `@PostConstruct`、作用域）
- AOP：代理（JDK/CGLIB）与常见坑（自调用）
- `@Transactional`：传播、失效场景（private、同类调用、异常类型）
- Spring MVC：过滤器/拦截器/DispatcherServlet 大致链路

**做什么**

1. 结合 `boot-social-demo`：写一段「**我为什么这样分层**」的解释（controller 薄、service 事务边界、mapper 只负责 SQL）。
2. 整理 8 道高频题（你自己答案）。

**验收**

- 能画出 **一次 HTTP 请求**到 controller 的简图（口头也行）。

---

## Day 242（周四）— MySQL & 数据相关（索引、事务、SQL 优化）

**学什么**

- 索引：**B+树直觉**、最左前缀、覆盖索引、`EXPLAIN` 看什么
- 事务：ACID、隔离级别与三类问题
- 慢查询思路：索引、回表、排序、分页深分页

**做什么**

1. 从你项目里挑 **1 条真实 SQL**（列表查询或 join），写：
   - 可能慢在哪里  
   - 你会加什么索引  
   - `EXPLAIN` 你怎么判断好坏  
2. 口述：「支付/订单」场景下如何保证 **订单与支付** 的一致（最终一致 / 幂等 / MQ）。

**验收**

- 你能用 **1 个例子**把索引讲明白（不等同于背定义）。

---

## Day 243（周五）— Redis / MQ / 微服务（结合你做过的）

**学什么**

- Redis：缓存穿透/击穿/雪崩（能讲定义 + 对应策略）
- MQ：至少一次投递、幂等消费、顺序消息（知道边界）
- 微服务：注册发现、网关、熔断限流、分布式 tracing（概念）

**做什么**

1. 用你项目语言复述：
   - **Redis**：你缓存了什么 key、TTL、怎么失效  
   - **MQ**：你发了什么事件、怎么幂等、失败怎么办  
   - **Nacos/Gateway/Sentinel**：各自解决什么问题  
2. 写「如果被问：为什么不用 Seata？」的一段话（结合 W34 笔记）。

**验收**

- 每个组件你能用 **一句话**说明 **trade-off**。

## Day 244（周六）— 场景题与系统设计（小步版，不写大论文）

**常见问法（选 3 个练）**

- 设计一个秒杀/抢券（你学个简化版：库存、防超卖、限流）
- 设计一个 Feed 流/发帖列表（分页、缓存、写扩散/读扩散只说概念）
- 设计一个文件上传（MinIO：key 设计、鉴权、直链/presigned）

**做什么**

1. 每个场景写 **10 行**：核心表、核心接口、关键风险、你怎么兜底  
2. 把其中 1 个映射到 `boot-social-demo` 或 `dual-system-ms` 的真实实现（有项目支撑最有说服力）。

**验收**

- 每个场景你能 **5 分钟**讲完，不停顿补组件清单。
