## 35周Java后端学习路径（优化版总纲）

适用人群：每天约 2 小时，目标是把 Java 后端“学精”（能写、能改、能讲、能上线）。

这份总纲的优化原则（你后面按周走就行）
- **工程化前置**：Git / Maven / 日志 / 单测尽早变成习惯，避免后期大返工。
- **Web先抓原理**：Servlet/Tomcat 学“HTTP与生命周期/Session/Filter”等核心，不在页面渲染技术上耗太久。
- **中间件与微服务项目驱动**：Redis/Nginx/Docker/微服务组件按“项目需要”逐个接入，保证学深而不是跑通清单。
- **每周必须有交付物**：至少一个可运行的小项目/模块，持续迭代同一条主线项目（从控制台 → REST API → 可部署）。

你现在的进度
- 你已经完成了第1–5周的核心训练（语法→数组方法→OOP→多态→异常/字符串）。

## 阶段一：Java零基础入门（第1–7周）

### 第1周（环境搭建 + 语法入门）
目标：会写最基本的控制台程序；会用断点调试；输入校验不崩。
交付物：菜单小程序（计算器/ATM二选一）+ 笔记。

### 第2周（循环 + 数组 + 方法）
目标：循环/数组/方法拆分写熟，能写“有点复杂的逻辑”。
交付物：数组移位与统计 + 学生成绩管理（控制台版）。

### 第3周（封装 + 继承）
目标：会设计类、构造器、封装规则；理解继承/重写/super。
交付物：学生管理 OOP 升级（更严格封装）+ 部门员工管理（继承练习）。

### 第4周（Object/final/注解 + 单例 + 多态 + 抽象类/接口 + 内部类）
目标：把“面向对象的高级用法”用到小案例里（接口、多态、单例、转型）。
交付物：单例（饿汉/懒汉）+ 接口多态案例（支付/图形二选一）+ 内部类练习。

### 第5周（异常 + 包装类 + 字符串）
目标：程序健壮性（异常体系）+ 字符串处理熟练 + 能把校验与错误处理模板化。
交付物：异常版安全计算器 + 文本统计（字符串项目）。

### 第6–7周（集合 + 泛型 + 多线程 + I/O + 综合案例）
目标：掌握“常用工具箱”：集合/泛型/线程/IO；能写一个小型并发+IO案例。
交付物：多线程下载器（或并发文件拷贝/日志采集器）+ 集合/泛型练习集。
说明（优化点）：这两周开始就把“代码组织”做规范：目录、包、工具类、异常处理风格统一。

## 阶段二：Java数据库开发与实战（第8–12周）

### 第8–9周（MySQL）
目标：把“表设计 + SQL + 事务 + 索引”打牢（后端核心竞争力之一）。
交付物：为你前面的学生/员工系统设计数据库（建表SQL + 常用查询SQL + 索引思考）。
说明（优化点）：SQL 不只要会写，还要会解释：为什么这样写、哪里会慢、怎么加索引。

### 第10周（JDBC + 连接池 + DBUtils）
目标：Java 代码稳定操作数据库（CRUD、分页、事务、预编译防注入）。
交付物：把“学生/员工管理”接入 MySQL（DAO 层 + 连接池 + 基础事务）。

### 第11周（Java高级编程：Maven/工厂/反射/Lambda/Stream）
目标：工程化与现代 Java 编程方式入门。
交付物：把项目改成 Maven 工程 + 引入日志 + 写最少量单测（从这周开始养成习惯）。
说明（优化点，重要）：Maven/日志/测试不要只“会用一次”，要变成后续每周都在用的默认配置。

### 第12周（MyBatis）
目标：用 MyBatis 提升数据库开发效率（Mapper、动态SQL、多表查询、分页、缓存概念）。
交付物：用 MyBatis 重写第10周的 DAO（同一业务两种实现对比，会非常长进）。

## 阶段三：从网页搭建入门Java Web（第13–20周）

### 第13–14周（HTML/CSS/JS）
目标：能看懂前端页面、能改样式、能写简单交互（为后续联调做准备）。
交付物：一个静态页面 + JS 小交互（表单校验/列表渲染）。
说明（优化点）：前端别追求“精通”，以“能联调 + 能改需求”为目标即可。

### 第15–16周（DOM/BOM/ES6 + 前端综合案例 + Vue3入门）
目标：掌握 DOM 操作与 ES6 基本语法；Vue3 入门能跑起来。
交付物：仿站页面 + Vue3 最小 demo（列表渲染、事件、组件化初体验）。

### 第17周（Java Web：Tomcat + Servlet）
目标：掌握 Web 的底层关键原理：请求/响应、Servlet 生命周期、URL映射、GET/POST。
交付物：2–3 个 Servlet 接口（返回文本/JSON），会用浏览器/工具访问并理解参数。
说明（优化点）：这一周的重点是“理解 HTTP 与容器”，不要在 JSP/页面渲染上耗太久。

### 第18周（Ajax/JSON + 正则校验 + Filter）
目标：前后端交互 + 通用能力（编码、过滤器链、适配）。
交付物：一个 Ajax 调用后端接口的页面 + 一个 Filter（统一编码/简单鉴权/日志）。

### 第19–20周（MyBatis实现OA系统实战 + Linux基础）
目标：把“需求→表设计→后端接口→页面联调”跑一遍。
交付物：OA 核心流程跑通（登录/请假/审批至少一条主线）+ 能在 Linux 上部署运行。
说明（优化点）：Linux 与部署要结合你的项目来学（目录、权限、打包、运行、日志定位）。

## 阶段四：从SSM到SpringBoot 项目实战（第21–28周）

### 第21–22周（Spring：IoC/DI/AOP/事务/JdbcTemplate）
目标：理解 Spring 的核心思想与使用方式，能写出可维护的分层结构。
交付物：把一个小业务迁移到 Spring 管理（Bean、AOP日志、声明式事务）。

### 第23周（Spring MVC：REST/参数绑定/跨域/拦截器）
目标：会写规范的 REST API（统一返回、异常处理、拦截器）。
交付物：把学生/员工管理做成 REST API（含分页/条件查询的雏形）。

### 第24周（SSM社交网站实战）
目标：做一个更贴近企业的项目结构（用户、内容、评论、缓存、审核等）。
交付物：至少完成“用户注册登录 + 列表/详情 + 评论/点赞”的主流程。

### 第25–26周（Spring Boot 3 + JDK21新特性 + MyBatis/MyBatis Plus）
目标：切到主流生产力栈：Spring Boot 3 + 更现代的 Java；掌握 Starter/整合链路。
交付物：一个 Spring Boot Web 应用（接口文档、参数校验、统一异常、日志、分层清晰）。

### 第27–28周（Redis7/Nginx/Docker/Git&GitLab + 项目重构升级）
目标：把项目变成“能上线、能协作、能优化”的形态。
交付物：项目 Docker 化部署 + Redis 缓存一个热点接口 + Nginx 反代 + Git 多分支协作一次。
说明（优化点）：Git 不等到这周才开始；但这周要把“团队协作流程”真正跑一遍。

## 阶段五：Java分布式与微服务实战（第29–34周）

### 第29–30周（分布式入门：Zookeeper/Dubbo/RabbitMQ/Seata/SaToken等）
目标：理解分布式问题与常用解法（注册发现、远程调用、消息、分布式事务、鉴权）。
交付物：在你的 Spring Boot 项目上接入 1–2 个组件并落地一个真实需求（例如登录鉴权 + 消息异步化）。
说明（优化点）：不要“组件清单式跑通”，要能讲清：为什么用、怎么用、代价是什么。

### 第31周（微服务入门：数据库集群/MyCat/任务调度/Minio/MongoDB）
目标：理解“数据与存储”在微服务里的常见方案。
交付物：选 1 个方向深挖并落地到项目（Minio 文件上传/下载，或 MongoDB 一个模块）。

### 第32周（微服务进阶：Nacos/Feign/Gateway/Sentinel）
目标：微服务最常用的一组组合拳（注册、调用、网关、限流熔断）。
交付物：拆 2 个最小服务 + Feign 调用 + Gateway 统一入口 + Sentinel 保护一个接口。

### 第33–34周（双系统项目实战）
目标：把微服务方案在真实业务中跑起来，形成“大项目经验”。
交付物：至少完成“用户/订单/支付/报告查询”等 1 条完整链路的后端实现与联调。

## 贯穿全程的“你只要照做就会越来越强”的规则

### 1）从现在开始就用 Git 管理你的学习代码
- 每天/每次完成一个小交付物就提交一次（提交信息写清“做了什么”）。

### 2）从第11周开始固定使用 Maven + 日志 + 单测
- 你后面每个项目都按这个标准起步，否则越写越乱。

### 3）每周必须有一个“可运行交付物”
- 哪怕很小，但必须能跑、能演示、能讲清楚设计。

### 4）中间件/微服务永远遵循“为项目服务”
- 先明确要解决的问题，再引入组件；不要为了学组件而学组件。
