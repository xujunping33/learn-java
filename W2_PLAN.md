## Day 8（while / do-while / for：选择最合适的循环）

学习要点
- while vs do-while（先判断 vs 先执行）
- for 的使用边界（计数最清晰）
- 循环条件与边界值：\(i < n\) vs \(i \le n\)

任务卡（70min编码）
- `LoopMenuDemo`：用 while 做“菜单循环”（复用你 W1 的经验）
- `InputUntilValid`：输入整数，直到在 0–100 范围才结束
- 练习题（至少 6 题）
  - 1..n 求和（for/while 各写一版）
  - 输入 n，打印 1..n 的偶数
  - 统计 10 次输入的最大值/最小值
  - 计算平均值（注意整除）
  - 计算 n 的阶乘（注意溢出：先用 long）
  - 反向输出一个整数（例如 123 -> 321）

验收标准
- 你能说清楚：同一题为何 for 更清晰/while 更合适
- 能避免死循环（每个循环都能指出“退出条件”）

学习要点
- 外层控制“行/轮次”，内层控制“列/细节”
- 复杂循环先画表格/找规律

任务卡（70min编码）
- `MultiplicationTable`：9x9 乘法表
- `PatternPrinter`：至少打印 3 种图形（星号三角形/菱形/空心矩形任选）

验收标准

## Day 10（break / continue：控制流程但不写乱）

学习要点
- break：跳出当前循环
- continue：跳过本轮
- 嵌套循环中 break 的作用范围（只跳一层）

任务卡（70min编码）
- `GuessNumberGame`：猜数字（给提示：大了/小了；限制次数；用 break 结束）
- `SkipBadInputStats`：输入若干整数，遇到非法输入/不在范围就 continue 跳过；统计有效数据平均值

验收标准
- 你能写出“用 continue 简化 if 嵌套”的例子

## Day 13（方法：参数/返回值/重载/拆分职责）

学习要点
- 方法签名：参数列表、返回值
- 什么时候该“拆方法”：重复代码、过长方法、可复用逻辑
- 重载：同名不同参

任务卡（70min编码）
- 把 Day11 的 `ArrayStats` 重构：统计逻辑拆成方法
  - `readIntInRange(...)`
  - `max(int[])` / `min(int[])` / `avg(int[])`
- `MethodOverloadDemo`：写 2–3 个重载方法（例如 `sum(int,int)`、`sum(int,int,int)`）

验收标准
- 你能把一个 80 行方法拆成 4–6 个更短的方法，并且程序仍然正确

## Day 14（大作业：学生成绩管理 StudentScoreManager）

目标：做一个“可持续迭代”的控制台小系统，为后面接 MySQL 做铺垫。

必须功能（第一版就要有）
- 菜单循环：新增学生 / 修改成绩 / 删除学生 / 查询学生 / 列表展示 / 统计分析 / 退出
- 学生数据：至少包含 `id`、`name`、`score`
- 输入校验：id 唯一、score 0–100、非法输入不崩溃
- 统计分析：平均分、最高分学生、最低分学生、不及格人数

强烈建议的最小分层（不强制，但会让你“学精”）
- `Student`：数据模型（字段 + 构造器 + getter/setter）
- `StudentService`：业务（增删改查、统计）
- `SafeInput`：安全输入（可以复用你 day6 的实现）
- `Main`：只负责菜单与调用 service

验收标准（完成即过关）
- 程序能运行 10 分钟不崩溃
- 任意乱输字符也不会直接异常退出（会提示并继续）
- 你能清晰讲出：为什么要分 `Main/Service/Model`

本周复盘（30min）
- 写出你最容易犯的 5 个数组/循环 bug（例如越界、边界条件、整除、变量初始化位置不对）
- 写出你本周最常复用的 3 个“方法模板”（例如：读取并校验输入、遍历统计、查找并返回索引）
